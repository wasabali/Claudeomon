## Kenney RPG Urban Pack (CC0)

Village, nature, and interior biome **main** and **gym** maps are rendered with tiles sourced from the
[Kenney RPG Urban Pack](https://kenney.nl/assets/rpg-urban-pack)
by [Kenney](https://kenney.nl). Dungeon-type children of those biomes use the custom generated biome
tilesets instead (see *Custom Pixel Art Tilesets — Biome Tilesets* below).

- **License:** [CC0 1.0 Universal](https://creativecommons.org/publicdomain/zero/1.0/)
  (no attribution required, but we credit them anyway)
- **Source:** https://kenney.nl/assets/rpg-urban-pack
- **File used:** `tilemap_packed.png` (27×18 tile grid, 16×16 px source)
- **Upscale applied:** 3× nearest-neighbour → 48×48 px → stored as `assets/maps/tilesets/kenney_urban.png` (1296×864 px)
- **Used by biomes (main/gym maps only):** `village` (localhost_town, azure_town), `nature` (production_plains, pipeline_pass), `interior` (three_am_tavern, bakery_interior, lab_interior, apartment_interior)

**Note:** The Kenney-sourced tiles above are currently rendered as pixel art in the
style of the Kenney RPG Urban Pack palette by `scripts/generate-tech-sprites.js`.
They should be replaced with the actual Kenney tiles when the pack is available.

**Important — upgrade workflow:** `npm run gen:tilesets` (and `npm run gen:tech-sprites`)
calls `generate-tech-sprites.js`, which **regenerates and overwrites** all 50 sprite PNGs
in `assets/tiles/sprites/` and `kenney_tech_office.png`. If you drop manually-sliced Kenney
tiles into `sprites/` before running either command, they will be overwritten.

To upgrade specific tiles with real Kenney art without losing them on the next `gen:tilesets` run:
1. Download `tilemap_packed.png` from https://kenney.nl/assets/rpg-urban-pack
2. Upscale 3× nearest-neighbour: `convert tilemap_packed.png -filter point -resize 300% upscaled.png`
3. Slice the 14 tiles listed above at 48×48 px each
4. Drop them into `assets/tiles/sprites/` with the matching `tile_NNN_name.png` filenames
5. Run `node scripts/compose-tech-tileset.js` (requires `npm install --save-dev canvas`)
   — this stitches only the sprites already present, leaving the rest from the current PNG
6. Update the painter functions for those tile IDs in `generate-tech-sprites.js` to match
   the real Kenney art so regeneration doesn't overwrite your work

## Custom Pixel Art Tilesets — Biome Tilesets

All four biome tilesets (`village.png`, `dungeon.png`, `nature.png`, `interior.png`)
are generated by `scripts/generate-tilesets.js` and stored in `assets/maps/tilesets/`.
Each is an 8-column × 4-row sprite sheet (32 tiles, 384×192 px) at 48×48 px per tile.
Matching `.tsx` Tiled tileset descriptors are generated alongside each PNG.

These are **custom pixel art** inspired by the GBA/DS Pokémon RPG visual style.
No third-party assets are used. All tiles are original works released under the
same licence as the rest of this repository.

| Tileset | Biome | Used by regions | Key tiles |
|---------|-------|-----------------|-----------|
| `village.png` | `village` | Dungeon-type children only (main maps use kenney_urban) | Grass, path, stone wall, wood fence, tree, water |
| `dungeon.png` | `dungeon` | jira_dungeon_1/2/3, node_modules_maze | Stone floor, stone wall, cracked wall, torch, barrel, door |
| `nature.png` | `nature` | Dungeon-type children only (main maps use kenney_urban) | Grass variants, dirt path, water, trees, rocks, cliffs |
| `interior.png` | `interior` | Dungeon-type children only (main maps use kenney_urban) | Wood floor, wall panels, furniture, door, window |

Run `npm run gen:tilesets` to regenerate all four biome PNGs and their `.tsx` files.

## Stub tileset

`stub_tiles.png` (5 tiles, same directory as maps) is the original placeholder
tileset used during early development. It is used as the base `firstgid=1` tileset
in maps that carry a supplemental second tileset (tech, void, wasteland, biome).
